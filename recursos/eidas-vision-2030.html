eIDAS: la Vision 2030 de internet en Europa

Internet como la plataforma segura sobre la que se desplegaran las relaciones y transacciones de ciudadanos y empresas con efectos legales reforzados. Donde quedan los «viejos» servicios eIDAS 2014?

La tesis: internet necesitaba una capa juridica

Internet se construyo como una red de comunicacion, no como una plataforma transaccional. No habia identidad digital, no habia firma electronica con valor legal, no habia forma de demostrar que un dato existia en un momento concreto. El Reglamento eIDAS (2014) fue el primer intento serio de Europa de corregir esto: anadir una capa de seguridad juridica a internet.

Los servicios que introdujo eIDAS 2014 — firma electronica, sellos de tiempo, entrega certificada, custodia — fueron pensados para una epoca de internet donde las transacciones digitales eran la excepcion, no la norma. Documentos PDF que se firmaban. Emails que se certificaban. Archivos que se custodiaban. Operaciones puntuales en un mundo aun analogico.

«eIDAS 2014 fue un exito regulatorio. Pero fue disenado para el internet de 2014: documentos, firmas, emails. El internet de 2030 es otra cosa.»

El internet de 2030: transaccional, automatizado, regulado

El internet que se esta construyendo es radicalmente diferente. Miles de operaciones digitales por segundo. Contratos que se ejecutan automaticamente. Sistemas de IA que toman decisiones. Tokens que representan activos reales. Identidades digitales verificables. Un internet donde cada interaccion tiene o deberia tener efectos juridicos.

eIDAS 2.0 responde a esta realidad con una vision mucho mas ambiciosa:

eIDAS 2014: servicios puntuales

  • Firma electronica de documentos
  • Sellos de tiempo de archivos
  • Entrega certificada de emails
  • Custodia de documentos

Pensado para operaciones manuales y puntuales

eIDAS 2030: infraestructura transaccional

  • Identidad digital universal (EUDI Wallet)
  • Atributos verificables (QEAA)
  • Registros distribuidos cualificados (QDLT)
  • Firma remota movil (MRQSCD)
  • Servicios de confianza como API

Infraestructura para miles de operaciones/segundo

Donde quedan los «viejos» servicios eIDAS 2014?

No desaparecen. Se transforman en infraestructura. La firma electronica deja de ser un acto puntual («firma este PDF») y se convierte en un microservicio API que se invoca miles de veces al dia. El sello de tiempo deja de aplicarse manualmente a un documento y se inyecta automaticamente en cada transaccion. La entrega certificada deja de ser un email especial y se convierte en un webhook certificado.

Esto es exactamente lo que hace Trust API (La Factoria): toma los servicios eIDAS «clasicos» y los expone como 4 microservicios REST consumibles a escala. Es el puente entre el eIDAS de 2014 (servicios puntuales) y el eIDAS de 2030 (infraestructura transaccional).

La vision de g-digital: Los servicios de confianza regulados son la infraestructura legal basica de las operaciones de una organizacion. No un add-on. No un proyecto puntual. Infraestructura. Igual que el cloud es infraestructura de computacion, eIDAS es infraestructura de seguridad juridica. Trust API es como lo implementas.
